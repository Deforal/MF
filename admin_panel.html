Инструкция админа: Чтобы изменить параметры в таблице, нажмите на нужную ячейку, введите новое значение и нажмите Enter — оно автоматически обновится в базе данных. Для удаления строки воспользуйтесь кнопкой «Удалить», расположенной напротив соответствующей записи. Для добавления нового товара используйте форму, расположенную в нижней части таблицы: корректно заполните все поля и нажмите «Добавить» — данные будут внесены в базу. Переключение между таблицами осуществляется нажатием на одну из четырех кнопок, расположенных в верхней части интерфейса. Названия колонок в основном интуитивно понятны (переведены с английского), однако следует учитывать следующие особенности: Amount_AVBL — количество на складе, Amount_Sold — количество проданных единиц; Second_price — цена со скидкой (оставляйте пустой, если скидка не предусмотрена); Group_id — отвечает за группировку однотипных товаров (например, различных вариантов одного и того же протеина с разными вкусами или объёмами), крайне важно, чтобы товары были сгруппированы логично — неправильная группировка (например, объединение протеина и креатина) может ввести пользователя в заблуждение. Значение Group_id связано с таблицей Descriptions — должно соответствовать колонке ID в ней. В таблице Descriptions следующие поля: Name — название группы, List — список преимуществ, Overview — общее описание, Benefits — польза, Suggest — рекомендации по применению. Все текстовые поля (кроме Name) заполняются сплошным текстом с разбивкой на абзацы — чтобы вставить отступ (перенос строки), используйте ### в конце нужного абзаца. В таблице Images: Product_id должен соответствовать колонке ID из таблицы товаров; URL — название изображения с расширением (например, img.webp); Selection — порядок отображения: 0 указывает, что данное изображение будет основным на карточке товара в каталоге (если ни одно изображение не имеет значения 0, карточка останется без фото). В таблице Users: колонка Role со значением 1 обозначает администратора — строго запрещается присваивать эту роль обычным пользователям, так как это может повлечь за собой нежелательные последствия для сайта. Кроме того, обратите особое внимание: в колонках Flavor, Size, Category и Type значения чувствительны к регистру. Это значит, что, например, указание «клубника» (с маленькой буквы) будет восприниматься как совершенно иной вкус, нежели «Клубника» (с заглавной), даже если визуально они идентичны. Рекомендуется соблюдать единый стиль написания и придерживаться уже существующего формата для предотвращения дублирования и некорректной группировки.
